下面把 **普通线性注意力、GatedDeltaNet、Mamba2、DeltaNet、Kimi-Linear** 放到同一张表里。
先给你一个判断框架：

* **普通线性注意力**：快，但“写入记忆”基本是**纯加法累积**，容易记忆污染。 ([arXiv][1])
* **DeltaNet**：把“纯加法写入”改成 **delta-rule 纠错式写入**，更擅长检索和精确改写。 ([arXiv][2])
* **Mamba2**：核心是 **selective / gated state update**，擅长“选择性遗忘和压缩状态”，并通过 SSD 框架做高效训练。 ([arXiv][3])
* **GatedDeltaNet**：把 **Mamba2 的 gate** 和 **DeltaNet 的 delta rule** 合起来，兼顾遗忘和精确写入。 ([arXiv][4])
* **Kimi-Linear**：以 **KDA（Kimi Delta Attention）** 为核心，把 GatedDeltaNet 再细化成 **channel-wise gate**，并采用混合式线性/全局注意力架构，更偏工程落地和超长上下文推理。 ([arXiv][5])

## 五者的计算流程图（Mermaid）

下面都是**教学版**流程图，抓的是核心计算逻辑，不是论文里的全部实现细节。

---

### 1) 普通线性注意力

```mermaid
flowchart TD
    A[输入 token x_t] --> B[线性投影得到 q_t k_t v_t]
    B --> C[特征映射 phi(q_t), phi(k_t)]
    C --> D[读取累计状态 S_{t-1}, z_{t-1}]
    D --> E[更新记忆<br/>S_t = S_{t-1} + phi(k_t)v_t^T]
    E --> F[更新归一化项<br/>z_t = z_{t-1} + phi(k_t)]
    F --> G[读出输出<br/>o_t = phi(q_t)^T S_t / phi(q_t)^T z_t]
```

它的关键点是：**把 softmax attention 改写成核特征映射后的线性递推**，所以可以边走边累计状态。 ([arXiv][1])

---

### 2) DeltaNet

```mermaid
flowchart TD
    A[输入 token x_t] --> B[得到 q_t k_t v_t]
    B --> C[读取旧状态 S_{t-1}]
    C --> D[查询当前 key 对应的旧值 old_t]
    D --> E[计算误差/增量<br/>delta_t = beta_t * (v_t - old_t)]
    E --> F[按 key_t 写回状态<br/>S_t = S_{t-1} + write(k_t, delta_t)]
    F --> G[用 q_t 从 S_t 读出 o_t]
```

它相对普通线性注意力最大的区别，是写入不再只是“加一个新外积”，而是**先看旧内容，再做纠错式更新**。 ([arXiv][2])

---

### 3) Mamba2

```mermaid
flowchart TD
    A[输入 token x_t] --> B[输入投影/卷积等预处理]
    B --> C[生成输入相关参数<br/>A_t B_t C_t 等]
    C --> D[读取旧状态 h_{t-1}]
    D --> E[选择性状态更新<br/>h_t = selective_SSM_update(h_{t-1}, x_t)]
    E --> F[状态读出 y_t]
    F --> G[输出投影]
```

Mamba2 的核心不是 key-value 映射式“外积写表”，而是**输入相关的 selective SSM 状态演化**；它在 SSD 框架下获得了更强的训练效率。 ([arXiv][3])

---

### 4) GatedDeltaNet

```mermaid
flowchart TD
    A[输入 token x_t] --> B[得到 q_t k_t v_t]
    B --> C[额外生成 gate g_t 与写入强度 beta_t]
    C --> D[读取旧状态 S_{t-1}]
    D --> E[先遗忘/衰减旧记忆<br/>S_t^- = gate(g_t, S_{t-1})]
    E --> F[读取当前 key 对应旧内容 old_t]
    F --> G[计算纠错增量<br/>delta_t = beta_t * (v_t - old_t)]
    G --> H[按 key_t 写回<br/>S_t = S_t^- + write(k_t, delta_t)]
    H --> I[用 q_t 从 S_t 读出 o_t]
```

这就是它最重要的思想：**先忘，再精写**。论文就是把 Mamba2 风格的 gating 和 DeltaNet 风格的 delta rule 合并。 ([arXiv][4])

---

### 5) Kimi-Linear（KDA 核心流程）

```mermaid
flowchart TD
    A[输入 token x_t] --> B[得到 q_t k_t v_t]
    B --> C[生成更细粒度的 channel-wise forget gate alpha_t]
    C --> D[生成写入强度 beta_t]
    D --> E[读取旧状态 S_{t-1}]
    E --> F[按通道衰减旧状态<br/>S_t^- = channel_gate(alpha_t, S_{t-1})]
    F --> G[读取当前 key 对应旧内容 old_t]
    G --> H[计算增量<br/>delta_t = beta_t * (v_t - old_t)]
    H --> I[按低秩/特化状态转移写回<br/>S_t = S_t^- + low_rank_write(k_t, delta_t)]
    I --> J[读出 o_t]
    J --> K[与部分全局注意力层交替组成混合架构]
```

KDA 被论文描述为 **GatedDeltaNet 的 refined version**：主要升级点是**更细粒度 gate** 和 **更适合硬件的状态转移参数化**。 ([arXiv][5])

---

## 一句话横向对照

你可以把这五者记成一条演化链：

```mermaid
flowchart LR
    A[普通线性注意力<br/>纯加性记忆累积]
    --> B[DeltaNet<br/>delta-rule纠错写入]
    --> C[GatedDeltaNet<br/>gate + delta rule]
    --> D[Kimi-Linear / KDA<br/>channel-wise gate + hybrid architecture]

    A --> E[Mamba2<br/>selective SSM / gating]
    E --> C
```

这条链最核心的逻辑是：

* **普通线性注意力**解决“算得快”
* **DeltaNet**解决“写得准”
* **Mamba2**解决“忘得好”
* **GatedDeltaNet**解决“既能忘、又能准写”
* **Kimi-Linear**解决“把这套东西做得更细粒度、并更适合长上下文推理部署” ([arXiv][1])
